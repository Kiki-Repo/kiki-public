Privacy Policy

Effective date: March 22, 2026 · Last updated: March 25, 2026

Kiki is a voice-controlled browser assistant distributed as a Chrome extension. This policy explains what data Kiki accesses, how it is processed, and what leaves your device.

The short version:

Wake word detection runs entirely on your device. Voice audio is transcribed in real time and immediately discarded. Page snapshots exist only during a command and are never stored. The only data we collect is anonymous usage signals like "a click succeeded" — never what you clicked, what you said, or which page you were on.

Who we are

Kiki is developed and maintained by Han Lee. For privacy questions, contact hanlee4g@gmail.com or open an issue on the project's GitHub repository.

1. Wake word detection is local

The wake word "Kiki" is detected entirely in your browser using a local machine-learning model (ONNX Runtime WebAssembly). No audio leaves your device until you actively give a voice command. The model runs in a sandboxed offscreen document within the extension — there is no server round-trip for wake word detection.

2. Voice commands are processed and discarded

When you give a voice command, audio is streamed to the Kiki server, which proxies it to Deepgram for speech-to-text transcription. The resulting text is forwarded to an AI model for action planning. Neither the audio nor the transcript is stored — they exist only for the duration of the request and are discarded immediately after processing.

The server forwards voice transcripts and page context to one of the following AI providers, depending on configuration:

  • Google Gemini (default) — API Terms of Service
  • Anthropic Claude — Privacy Policy
  • OpenAI — Privacy Policy

No direct communication occurs between the extension and these AI services. All data flows through the Kiki server middleware, which acts as a stateless proxy. Data sent to third-party providers is governed by their respective policies linked above.

3. Page snapshots are temporary

To understand what is on your screen, Kiki reads interactive elements on the current page (buttons, links, form fields, headings) and creates a structured accessibility snapshot. This snapshot:

  • Exists only in memory for the duration of your command
  • Is never written to disk, logged, or persisted on the server
  • Contains element labels and roles — not sensitive page content such as passwords or personal data displayed on screen

4. Anonymous telemetry

Kiki collects anonymous structural telemetry to help identify and fix broken functionality. We see that something worked or did not — never what you clicked, what you said, or what page you were on.

The following events are collected:

EventData included
Extension installedNone
Session startedNone
Command issuedAction type (e.g. "click", "scroll")
Command succeeded / failedAction type, error category (if failed)
Session endedTotal command count
Onboarding phase completedPhase name (e.g. "mic", "demos")

Each installation generates a random anonymous ID (UUID) stored locally. This ID is not linked to your identity in any way. No personal information, URLs, page content, or command transcripts are included in telemetry events.

5. Voluntary feedback

The chat panel includes an optional feedback button. When you choose to send feedback, the following data from recent command sessions may be included:

  • Task descriptions (what you asked Kiki to do)
  • Action types and outcomes (e.g. "click succeeded")
  • The starting URL of the session
  • An optional note you write

You see exactly which sessions will be included before sending and can deselect any you do not want to share. Feedback submission is entirely opt-in and voluntary.

6. Data stored on your device

The extension stores the following locally via chrome.storage.local. None of this data is synced to the cloud or shared with third parties:

  • Server URL configuration
  • Keyboard shortcut preference
  • Chat theme preference (dark / light)
  • Anonymous telemetry client ID (random UUID)
  • Recent command history (up to 50 sessions, for the voluntary feedback feature)
  • Onboarding completion status

7. Data retention

  • Audio and transcripts: Zero retention. Processed in real time and discarded immediately.
  • Page snapshots: Zero retention. Held in memory only during command execution.
  • Telemetry events: Retained on the server in aggregated, anonymous form. Individual events cannot be linked to a person.
  • Voluntary feedback: Retained for debugging purposes. May be deleted upon request.
  • Local data: Persists on your device until you uninstall the extension or clear extension data.

8. Data security

  • All communication between the extension and the Kiki server uses HTTPS (TLS encryption in transit).
  • API keys for third-party services (Deepgram, AI providers) are stored server-side only — never in the extension code or on your device.
  • The server runs on Google Cloud Run with automatic scaling and isolation.
  • The extension does not use cookies, local storage (browser localStorage), or any cross-site tracking mechanisms.

9. Permissions

The extension requests these Chrome permissions:

PermissionWhy it is needed
offscreenRun wake word detection and audio capture in a background document (required because MV3 service workers cannot access getUserMedia).
storageSave your preferences (server URL, hotkey, theme) locally on your device.
tabsRead the current page for accessibility snapshots and support multi-tab voice commands ("open a new tab", "switch to Gmail").
webNavigationDetect when page navigation completes so Kiki can continue multi-step tasks after a page change.
Host permissions (all URLs)Inject content scripts on any website so voice commands work immediately without requiring a click on the extension icon first. No page data is exfiltrated.

10. Children's privacy

Kiki is not directed at children under 13. We do not knowingly collect personal information from children. If you believe a child has provided personal information through the extension, please contact us and we will delete it promptly.

11. International data transfers

The Kiki server is hosted on Google Cloud in the United States (us-west1 region). Audio, transcripts, and page snapshots may be processed in the United States during command execution. Third-party AI providers may process data in additional jurisdictions as described in their own privacy policies.

12. Your rights

  • Delete local data: Uninstall the extension or go to chrome://extensions, click "Details" on Kiki, and select "Clear data." This removes all locally stored preferences, command history, and the anonymous telemetry ID.
  • Delete server-side data: Because telemetry uses a random UUID with no link to your identity, server-side telemetry records cannot be associated with you after you clear local data. If you submitted voluntary feedback and want it removed, contact us with the approximate date and we will delete it.
  • Opt-out of telemetry: Telemetry is currently on by default during beta. A toggle to disable it will be available in a future release.

13. Changes to this policy

If we make material changes to this policy, we will update the "Last updated" date at the top and, for significant changes, include a notice in the extension's update notes on the Chrome Web Store. Continued use of the extension after changes constitutes acceptance of the revised policy.
